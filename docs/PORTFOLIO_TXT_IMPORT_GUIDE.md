# TXT 文件格式支持 - 技术说明

## 🎯 功能概述

系统现已支持导入 **.txt 格式**的交易记录文件，自动识别并解析多种文本分隔格式。

## 📋 支持的 TXT 格式

### 1. 制表符分隔 (TSV) - 最常用

```txt
证券代码	成交日期	买卖方向	成交价格	成交数量
600519	2024-01-01	买入	1800.00	100
600520	2024-01-02	卖出	1850.00	50
```

**特点：**
- ✅ 券商导出最常见格式
- ✅ 列对齐清晰
- ✅ 不易与内容冲突

### 2. 逗号分隔 (CSV in TXT)

```txt
证券代码,成交日期,买卖方向,成交价格,成交数量
600519,2024-01-01,买入,1800.00,100
600520,2024-01-02,卖出,1850.00,50
```

**特点：**
- ✅ 与 CSV 格式相同
- ✅ 通用性强
- ⚠️ 如果字段包含逗号需要引号包裹

### 3. 空格分隔

```txt
证券代码 成交日期 买卖方向 成交价格 成交数量
600519 2024-01-01 买入 1800.00 100
600520 2024-01-02 卖出 1850.00 50
```

**特点：**
- ✅ 人类可读性好
- ⚠️ 字段不能包含空格
- ⚠️ 多个空格会被视为一个分隔符

### 4. 固定宽度

```txt
证券代码  成交日期    买卖方向  成交价格  成交数量
600519    2024-01-01  买入      1800.00   100
600520    2024-01-02  卖出      1850.00   50
```

**特点：**
- ✅ 老式系统常用
- ✅ 列对齐美观
- ⚠️ 需要自动检测列宽

## 🔧 技术实现

### 1. 编码检测

TXT 文件可能使用多种编码，系统会依次尝试：

```python
for encoding in ('utf-8', 'gbk', 'gb18030', 'utf-8-sig'):
    try:
        text = content.decode(encoding)
        break
    except UnicodeDecodeError:
        continue
```

**支持的编码：**
- UTF-8（国际标准）
- GBK（简体中文 Windows）
- GB18030（中国国家标准）
- UTF-8-SIG（带 BOM 的 UTF-8）

### 2. 分隔符检测策略

系统按以下顺序尝试解析：

```
Strategy 1: 制表符 (\t)
    ↓ 失败
Strategy 2: 逗号 (,)
    ↓ 失败
Strategy 3: 空白字符 (\s+)
    ↓ 失败
Strategy 4: 自动检测 (sep=None)
    ↓ 失败
报错并提供详细诊断信息
```

### 3. 智能验证

每种策略都会验证：
- ✅ DataFrame 非空
- ✅ 列数 > 1（排除单列数据）
- ✅ 成功解析的行数合理

## ⚠️ 注意事项

### 1. 表头行

**建议包含表头：**
```txt
证券代码	成交日期	买卖方向	价格	数量  ← 表头
600519	2024-01-01	买入	1800	100     ← 数据
```

**如果没有表头：**
- 系统会使用 Column 0, Column 1... 作为列名
- 可能需要手动映射列含义

### 2. 空行处理

```txt
证券代码	成交日期	买卖方向
600519	2024-01-01	买入
                    ← 空行会被跳过
600520	2024-01-02	卖出
```

系统会自动跳过空行和只包含空白字符的行。

### 3. 特殊字符

**字段包含分隔符时需要转义：**

```txt
# 错误：字段包含制表符
备注	这里	有	问题

# 正确：用引号包裹
备注	"这里	有	问题"
```

### 4. 编码问题

**Windows 记事本保存的文件：**
- 默认可能是 ANSI (GBK) 编码
- 建议选择 "UTF-8" 编码保存

**推荐工具：**
- VS Code（可显示和转换编码）
- Notepad++（编码菜单）
- Excel（另存时选择编码）

## 🔍 故障排查

### 问题 1：所有行都被跳过

**可能原因：**
- 没有必需的列（代码、日期、方向、价格、数量）
- 列名无法识别
- 数据格式错误

**解决方法：**
1. 检查是否包含必需列
2. 查看错误信息中的列名列表
3. 确保列名与支持的列名匹配

### 问题 2：编码错误

**错误信息：**
```
无法解码TXT文件
尝试的编码：utf-8, gbk, gb18030, utf-8-sig
```

**解决方法：**
1. 用文本编辑器打开文件
2. 查看当前编码
3. 另存为 UTF-8 或 GBK
4. 重新上传

### 问题 3：只有一列数据

**可能原因：**
- 分隔符不正确
- 文件实际上是单列数据

**解决方法：**
1. 检查分隔符是否正确（TAB、逗号、空格）
2. 在文本编辑器中查看原始内容
3. 转换为标准格式后重新上传

### 问题 4：列数不一致

**错误：**
```
某些行的列数与其他行不同
```

**解决方法：**
1. 检查每行的字段数是否一致
2. 确保没有缺失或多余的字段
3. 处理包含分隔符的字段（用引号包裹）
