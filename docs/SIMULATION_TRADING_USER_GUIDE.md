## 功能概述

模拟交易系统是一个完整的股票模拟交易平台，支持：

- ✅ **账户管理** - 创建和管理多个模拟账户
- ✅ **交易执行** - 模拟买入/卖出股票
- ✅ **交易建议** - AI 生成的网格交易策略
- ✅ **调度器** - 自动化定时任务（每日建议、复盘报告）
- ✅ **持仓跟踪** - 实时查看持仓和盈亏
- ✅ **T+1 规则** - 遵守 A 股交易制度

## 快速开始

### 1. 访问页面

打开浏览器访问：
```
http://localhost:5173/simulation
```

或使用 IP 地址：
```
http://127.0.0.1:5173/simulation
```

### 2. 首次使用

1. **自动加载账户** - 页面会自动加载所有模拟账户
2. **选择账户** - 从下拉列表选择一个账户（或自动选中第一个）
3. **查看快照** - 右侧显示账户持仓详情

### 3. 创建新账户（如需要）

> **注意**: 账户创建功能已移至 `/portfolio` 页面

访问 `http://localhost:5173/portfolio` 创建新账户，选择账户类型为 `simulation`。

## 账户管理

### 选择账户

1. 在左侧"账户选择"区域找到下拉列表
2. 点击下拉框，显示所有可用账户
3. 选择目标账户
4. 页面自动更新右侧持仓详情和顶部统计卡片

### 账户信息显示

**顶部统计卡片**:
- **可用资金** - 当前可用于交易的现金
- **总资产** - 可用资金 + 持仓市值
- **浮动盈亏** - 未实现盈亏（绿色=盈利，红色=亏损）
- **调度器** - 显示调度器运行状态

**持仓详情**:
- 账户名称、市场、基础货币
- 可用资金、总市值、总权益
- 实现盈亏、未实现盈亏
- 持仓列表（股票代码、数量、成本、盈亏）

### 刷新账户

点击"刷新状态"按钮更新：
- 账户列表
- 账户快照
- 调度器状态

## 交易执行

### 执行交易步骤

1. **点击"执行交易"按钮**
   - 确保已选择账户
   - 如果未选择账户，会提示"请先选择账户"

2. **填写交易表单**

   | 字段 | 说明 | 示例 |
   |------|------|------|
   | 股票代码 | 股票交易代码 | `600519` |
   | 买卖方向 | BUY=买入, SELL=卖出 | 买入 ↑ / 卖出 ↓ |
   | 价格 | 交易单价（元） | `100.00` |
   | 数量 | 交易股数（100的倍数） | `100` |

3. **提交交易**
   - 点击"确定"按钮
   - 等待处理
   - 成功后显示"交易成功: ID=xxx"

4. **查看结果**
   - 对话框自动关闭
   - 持仓详情自动刷新
   - 统计卡片更新

### 交易规则

**A 股交易制度**:
- ✅ T+1 规则 - 当日买入次日可卖
- ✅ 最小交易单位 - 100 股（1 手）
- ✅ 价格精度 - 2 位小数
- ️ 手续费 - 当前为 0（未来版本将支持）
- ⚠️ 税费 - 当前为 0（未来版本将支持）

**交易验证**:
- 卖出时检查可用持仓（不包含当日买入）
- 买入时检查可用资金
- 价格和数量格式验证

### 交易记录

交易记录保存在：
1. **Portfolio 系统** - 完整交易历史
2. **账户快照** - 实时更新持仓
3. **现金流水** - 资金变动记录

## 交易建议

### 生成交易建议

1. **点击"生成交易建议"按钮**
   - 确保已选择账户
   - 加载状态显示（按钮变灰）

2. **等待 API 响应**
   - 系统调用 AI 分析引擎
   - 分析市场情绪
   - 生成网格订单

3. **查看建议**
   - 自动弹出对话框
   - 显示分析结果

### 交易建议内容

**基本信息**:
- 股票代码
- 当前价格
- 市场情绪（看涨/看跌/中性）
- 网格订单数

**网格订单表格**:

| 类型 | 说明 | 颜色 |
|------|------|------|
| 建仓 | 买入开仓 | 蓝色 |
| 止盈 | 盈利卖出 | 绿色 |
| 止损 | 亏损卖出 | 红色 |

| 方向 | 说明 | 颜色 |
|------|------|------|
| 买入 ↑ | 建仓订单 | 绿色 |
| 卖出 ↓ | 平仓订单 | 红色 |

**交易建议文本**:
- AI 生成的文字建议
- 包含策略说明
- 风险提示

### 使用建议

**建议用途**:
- ✅ 作为交易参考
- ✅ 了解市场情绪
- ✅ 规划网格交易策略
- ⚠️ 不保证盈利
- ⚠️ 需结合个人判断

**当前限制**:
- 股票代码暂时硬编码为 `600519`
- 未来版本将支持用户输入股票代码

## 调度器管理

### 调度器功能

调度器是一个自动化任务系统，包含 3 个定时任务：

| 任务名称 | 执行时间 | 功能说明 |
|----------|----------|----------|
| 生成每日交易建议 | 每日 09:00 | 开盘前生成交易建议 |
| 盘中检查 | 每日 11:30 | 上午收盘前检查持仓 |
| 生成每日复盘报告 | 每日 15:30 | 收盘后生成复盘报告 |

### 调度器状态

**状态提示框**:
- 🟢 **运行中** (绿色) - 调度器正在运行
- 🟡 **已停止** (黄色) - 调度器未运行

**按钮状态**:
- 运行中 → 显示"停止调度器"（默认按钮样式）
- 已停止 → 显示"启动调度器"（主要按钮样式）

### 操作指南

#### 启动调度器

1. 确认状态为"已停止"
2. 点击"启动调度器"按钮
3. 等待成功提示
4. 状态自动刷新为"运行中"

#### 停止调度器

1. 确认状态为"运行中"
2. 点击"停止调度器"按钮
3. 等待成功提示
4. 状态自动刷新为"已停止"

#### 触发每日建议

1. 点击"触发每日建议"按钮
2. 立即执行"生成每日交易建议"任务
3. 不等待定时任务触发
4. 适用于测试或手动生成建议

#### 刷新状态

1. 点击"刷新状态"按钮
2. 同时刷新：
   - 调度器状态
   - 账户列表
   - 账户快照
3. 确保显示最新数据

## 持仓详情

### 查看持仓

**持仓信息**:
- 股票代码
- 持仓数量（股）
- 成本价
- 当前价（快照时）
- 浮动盈亏

**持仓卡片**:
```
┌─────────────────┐
│ 600519          │
│ 100 股          │
│ 成本: ¥100.00   │
│ 盈亏: ¥500.00   │
└─────────────────┘
```

### 持仓计算

**可用持仓** = 总持仓 - 冻结持仓

**T+1 规则**:
- 当日买入 → 次日可卖
- 历史持仓 → 立即可卖

**持仓更新时机**:
- 交易执行后立即更新
- 页面刷新时更新
- 定时刷新（未来功能）

### 空持仓状态

当账户没有持仓时：
- 显示"暂无持仓"空状态
- 提供"执行交易"快捷入口

## 常见问题

### Q1: 为什么点击"执行交易"提示"请先选择账户"？

**原因**: 未选择账户或账户列表为空

**解决方法**:
1. 检查是否已从下拉列表选择账户
2. 如果列表为空，点击"刷新状态"按钮
3. 如果仍无账户，需先在 `/portfolio` 页面创建模拟账户

### Q2: 交易成功后持仓没有更新？

**原因**: 快照未刷新

**解决方法**:
1. 点击"刷新状态"按钮
2. 或手动刷新浏览器页面
3. 检查交易是否真的成功（查看控制台日志）

### Q3: 调度器状态显示不准确？

**原因**: 状态未同步

**解决方法**:
1. 点击"刷新状态"按钮
2. 检查后端日志确认调度器真实状态
3. 查看控制台日志：`[checkSchedulerStatus] Status: {...}`

### Q4: 生成交易建议很慢？

**原因**: AI 分析需要时间

**解决方法**:
1. 等待加载状态完成（按钮变灰）
2. 查看控制台日志确认进度
3. 检查后端日志是否有错误

### Q5: 启动调度器失败？

**可能原因**:
- 调度器已在运行
- 后端服务未启动
- API 端点不可用

**解决方法**:
1. 检查状态是否为"运行中"
2. 刷新状态后重试
3. 检查后端日志：`logs/api_server_YYYYMMDD.log`

### Q6: 如何查看调试信息？

**方法**:
1. 按 `F12` 打开浏览器开发者工具
2. 切换到"Console"标签
3. 查看带 `[` 前缀的日志：
   ```
   [handleOpenTrade] selectedAccount: {...}
   [handleExecuteTrade] Form values: {...}
   [handleGetSuggestion] Suggestion data: {...}
   ```

### Q7: 交易建议对话框没有内容？

**原因**: 数据未正确加载

**解决方法**:
1. 查看控制台日志确认 API 响应
2. 检查 `suggestionData` 是否为 null
3. 查看后端日志是否有错误

### Q8: 如何重置调度器？

**步骤**:
1. 点击"停止调度器"
2. 等待确认已停止
3. 点击"启动调度器"
4. 验证状态更新

## 技术信息

### 前端技术栈

- **React** - UI 框架

| 方法 | 路径 | 说明 |
|------|------|------|

## 最佳实践

### 日常操作流程

1. **开盘前 (09:00 前)**
   - 启动调度器
   - 查看每日交易建议
   - 规划当日交易策略

2. **交易中 (09:30 - 15:00)**
   - 根据建议执行交易
   - 监控持仓变化
   - 注意 T+1 规则

3. **收盘后 (15:30 后)**
   - 查看复盘报告
   - 分析当日盈亏
   - 调整次日策略

### 账户管理

- ✅ 定期刷新账户状态
- ✅ 检查调度器运行状态
- ✅ 查看交易记录
- ⚠️ 不要频繁启动/停止调度器
- ️ 不要在交易时段修改调度器配置

### 风险控制

- ⚠️ 模拟交易仅供参考，不构成投资建议
- ⚠️ 设置止损止盈
- ️ 控制单笔交易金额
- ️ 定期检查账户余额
- ⚠️ 不要过度交易
