# 券商交易记录导入 - Excel 格式支持

## ✨ 新功能

系统现已支持导入 **Excel 格式** (.xls/.xlsx) 的券商交易记录，不再仅限于 CSV 格式。

## 📋 支持的格式

| 格式 | 扩展名 | 说明 | 引擎 |
|------|--------|------|------|
| CSV | `.csv` | 逗号分隔值文件 | pandas CSV parser |
| Excel (新版) | `.xlsx` | Office Open XML | openpyxl |
| Excel (旧版) | `.xls` | Excel 97-2003 | xlrd |
| **HTML 表格** | `.xls/.html` | **券商导出的 HTML 表格** | **lxml/html5lib** |
| **文本文件** | `.txt` | **制表符/空格分隔文本** | **pandas TSV parser** |

## 🔧 安装依赖

如果您还没有安装 Excel 读取库，请运行：

```bash
pip install openpyxl xlrd lxml html5lib
```

或者重新安装所有依赖：

```bash
pip install -r requirements.txt
```

## 📤 使用方法

### 方法 1：网页界面上传

1. 访问 http://127.0.0.1:8000/portfolio
2. 选择账户
3. 点击"导入交易记录"
4. 选择文件（支持 .csv/.xls/.xlsx）
5. 选择券商类型（华泰/中信/招商等）
6. 点击"解析"预览数据
7. 确认无误后点击"提交"

### 方法 2：API 调用

```bash
curl -X POST "http://127.0.0.1:8000/api/v1/portfolio/imports/csv/parse" \
  -F "broker=huatai" \
  -F "file=@trades.xlsx"
```

## 📊 Excel 文件格式要求

### 基本结构

Excel 文件应该包含以下列（列名可以略有不同，系统会自动识别）：

| 必需列 | 常见列名 | 说明 | 示例 |
|--------|---------|------|------|
| 证券代码 | 证券代码、股票代码、代码 | 6位数字代码 | 600519 |
| 成交日期 | 成交日期、发生日期、日期 | YYYY-MM-DD 格式 | 2024-01-01 |
| 买卖方向 | 买卖方向、买卖标志、操作 | 买入/卖出 | 买入 |
| 成交价格 | 成交价格、成交均价、价格 | 数值 | 1800.00 |
| 成交数量 | 成交数量、数量、成交股数 | 整数 | 100 |

| 可选列 | 常见列名 | 说明 |
|--------|---------|------|
| 手续费 | 手续费、佣金、交易费 | 数值 |
| 印花税 | 印花税、税费 | 数值 |
| 成交编号 | 成交编号、合同编号、流水号 | 字符串 |
| 币种 | 币种、货币 | CNY/HKD/USD |

### 从券商导出的 Excel 文件

大多数券商 APP 或网站导出的 Excel 文件都可以直接使用，系统会自动：

1. ✅ 识别不同的列名（如"证券代码"、"股票代码"、"代码"都能识别）
2. ✅ 跳过空行和无效行
3. ✅ 处理多种日期格式
4. ✅ 自动识别买卖方向（买入/卖出/B/S）

## 🎯 支持的券商

系统内置了以下券商的列名映射规则：

### 华泰证券 (huatai)
- 识别列名：成交日期、证券代码、买卖标志、成交均价、成交数量等

### 中信证券 (citic)
- 识别列名：发生日期、证券代码、买卖方向、成交价格、成交数量等

### 招商银行 (cmb)
- 识别列名：日期、证券代码、交易方向、成交价、成交股数等

### 通用格式
如果券商不在上述列表中，系统会尝试使用通用列名匹配。

## 💡 最佳实践

### 1. 导出时选择正确的格式

从券商 APP 或网站导出时：
- ✅ 优先选择 **Excel (.xlsx)** 格式
- ✅ 其次选择 **CSV** 格式
- ⚠️ 避免选择 PDF 或图片格式

### 2. 检查数据完整性

导入前快速检查：
- 是否有空行
- 日期格式是否正确
- 股票代码是否完整（6位）
- 买卖方向是否清晰

### 3. 清理不必要的工作表

如果 Excel 文件包含多个工作表：
- 确保交易数据在**第一个工作表**
- 或删除其他无关的工作表

### 4. 保持原始格式

不要手动修改券商导出的文件格式，直接上传即可。

## ❓ 常见问题

### Q1: 上传 .xls 文件时报错？

**A:** 需要安装 `xlrd` 库：
```bash
pip install xlrd>=2.0.0
```

### Q2: Excel 文件有多个工作表怎么办？

**A:** 系统会自动读取**第一个工作表**。请确保交易数据在第一个工作表中。

### Q3: 日期格式不支持？

**A:** 系统支持多种日期格式：
- `2024-01-01`
- `2024/01/01`
- `20240101`
- `2024年01月01日`

如果仍然无法识别，请在 Excel 中将日期列格式化为标准格式。

### Q4: 某些行被跳过了？

**A:** 可能的原因：
- 缺少必需字段（代码、日期、方向、价格、数量）
- 买卖方向无法识别（必须是"买入"或"卖出"）
- 价格为 0 或负数
- 数量为 0 或负数

系统会在解析结果中显示跳过的行数和错误信息。

### Q5: 中文列名无法识别？

**A:** 系统已内置常见的中文列名映射。如果仍无法识别，可以：
1. 将列名改为英文（code, date, side, price, quantity）
2. 或在 GitHub 提交 issue，我们会添加新的映射规则

### Q6: 华泰证券导出的 .xls 文件报错？

**A:** 华泰证券等券商导出的 .xls 文件实际上是 **HTML 表格格式**，而非真正的 Excel 文件。

**解决方案：**
- ✅ **直接上传**：系统现已支持自动识别并解析 HTML 表格格式
- ✅ **安装依赖**：确保安装了 `lxml` 和 `html5lib`
  ```bash
  pip install lxml html5lib
  ```
- ✅ **另存为真正Excel**：用 Excel 打开后，另存为 .xlsx 格式

### Q7: 如何导入 .txt 格式的交易记录？

**A:** 系统支持多种 TXT 文件格式：

**支持的 TXT 格式：**
1. **制表符分隔 (TSV)** - 最常见
   ```
   证券代码\t成交日期\t买卖方向\t成交价格\t成交数量
   600519\t2024-01-01\t买入\t1800.00\t100
   ```

2. **逗号分隔 (CSV in TXT)**
   ```
   证券代码,成交日期,买卖方向,成交价格,成交数量
   600519,2024-01-01,买入,1800.00,100
   ```

3. **空格分隔**
   ```
   证券代码 成交日期 买卖方向 成交价格 成交数量
   600519 2024-01-01 买入 1800.00 100
   ```

**使用方法：**
- ✅ 直接上传 .txt 文件
- ✅ 系统会自动检测分隔符
- ✅ 支持 UTF-8、GBK、GB18030 编码

## 🔍 故障排查

### 问题：上传后显示"无法解析文件"

**可能原因：**
1. 文件不是有效的 Excel/CSV 格式
2. 文件已损坏
3. 文件格式与扩展名不匹配（如 .xlsx 实际是 .xls）

**解决方法：**
1. 用 Excel 打开文件，另存为新的 .xlsx 文件
2. 确保文件扩展名与实际格式一致
3. 尝试转换为 CSV 格式后重新上传

### 问题：所有行都被跳过

**可能原因：**
1. 列名完全不匹配
2. 数据格式错误

**解决方法：**
1. 检查列名是否与支持的列名相似
2. 查看错误信息中的具体提示
3. 参考上面的"示例数据"调整格式
