# CSV 导入问题排查指南

## 问题描述

在 http://127.0.0.1:8000/portfolio 页面导入券商 CSV 时出现错误：

```
Error tokenizing data. C error: Expected 1 fields in line 11, saw 2
```

## 问题原因

这个错误表示 CSV 文件的第 11 行格式不正确：
- **期望**：1 个字段（列）
- **实际**：2 个字段（列）

这通常由以下原因导致：

### 1. 分隔符不一致
CSV 文件中某些行使用了不同的分隔符（如逗号、制表符混用）。

**示例：**
```csv
证券代码,成交日期,买卖方向,成交价格,成交数量
600519,2024-01-01,买入,1800.00,100
600520,2024-01-02,卖出,1850.00,50
错误行,这里有多余内容  ← 这一行有问题
```

### 2. 空行或特殊字符
文件中包含空行、注释行或不可见字符。

### 3. 引号不匹配
某些字段包含逗号但没有正确用引号包裹。

**示例：**
```csv
"贵州茅台",2024-01-01,"买入,限价单",1800.00,100  ← 引号使用不当
```

### 4. 编码问题
文件使用了非标准编码（GBK、UTF-8-BOM 等）。

## 解决方案

### ✅ 已实施的改进

系统现在已经增强了 CSV 解析能力，会自动尝试以下策略：

1. **多编码尝试**：依次尝试 UTF-8-SIG、GBK、GB18030
2. **容错解析**：跳过有问题的行（`on_bad_lines='skip'`）
3. **自动检测分隔符**：智能识别逗号、制表符等
4. **灵活引擎**：使用 Python 引擎而非 C 引擎，更宽容

### 🔧 手动修复 CSV 文件

如果仍然遇到问题，请按以下步骤检查和修复：

#### 步骤 1：用文本编辑器打开 CSV

推荐使用：
- **VS Code**（可显示隐藏字符）
- **Notepad++**（可查看编码）
- **Excel**（另存为时选择正确的格式）

#### 步骤 2：检查第 11 行附近

找到报错的行（第 11 行），检查：

1. **列数是否一致**
   ```csv
   # 正确：每行都有 5 列
   证券代码,成交日期,买卖方向,成交价格,成交数量
   600519,2024-01-01,买入,1800.00,100
   
   # 错误：这一行只有 2 列
   600520,2024-01-02
   ```

2. **是否有空行**
   ```csv
   证券代码,成交日期,买卖方向,成交价格,成交数量
   600519,2024-01-01,买入,1800.00,100
   
   ← 删除这个空行
   600520,2024-01-02,卖出,1850.00,50
   ```

3. **是否有特殊字符**
   - 删除行尾的空格
   - 删除不可见字符（如 `\r`, `\t`）

#### 步骤 3：统一分隔符

确保所有行使用相同的分隔符：

```csv
# 使用逗号（推荐）
证券代码,成交日期,买卖方向,成交价格,成交数量

# 或使用制表符（TAB）
证券代码	成交日期	买卖方向	成交价格	成交数量
```

#### 步骤 4：正确处理包含逗号的字段

如果字段内容包含逗号，必须用双引号包裹：

```csv
# 错误
备注,买入,普通交易

# 正确
备注,"买入,普通交易"
```

#### 步骤 5：保存为正确的编码

- **推荐**：UTF-8 with BOM (UTF-8-SIG)
- **备选**：GBK（适用于中文 Windows）

在 Excel 中：
1. 文件 → 另存为
2. 选择 "CSV UTF-8 (逗号分隔) (*.csv)"
3. 保存

### 💡 快速修复技巧

#### 方法 1：用 Excel 重新保存

1. 用 Excel 打开 CSV 文件
2. 检查数据是否正确显示
3. 文件 → 另存为 → 选择 "CSV UTF-8"
4. 重新上传

#### 方法 2：用 VS Code 清理

1. 用 VS Code 打开 CSV
2. 查看第 11 行是否有异常
3. 删除空行和多余空格
4. 保存为 UTF-8 编码

#### 方法 3：用 Python 脚本清理

```python
import pandas as pd

# 读取原始文件（容错模式）
df = pd.read_csv('original.csv', 
                 encoding='gbk', 
                 on_bad_lines='skip',
                 engine='python')

# 保存为标准 CSV
df.to_csv('cleaned.csv', 
          index=False, 
          encoding='utf-8-sig')

print(f"清理完成！共 {len(df)} 行数据")
```

## 常见错误及解决

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| `Expected 1 fields in line X, saw Y` | 第 X 行列数不一致 | 检查该行，补全或缺失的列 |
| `UnicodeDecodeError` | 编码不匹配 | 尝试 GBK 或 UTF-8-SIG 编码 |
| `EmptyDataError` | 文件为空或全是空行 | 删除空行，保留数据行 |
| `ParserError` | 分隔符混乱 | 统一使用逗号或制表符 |
